# 项目型费用管理体系

这是一个围绕“项目阶段”设计的费用管理案例。

重点不是只处理报销，而是从项目立项、预算启用、费用发生、审核控制到核销收尾，建立一套可追踪、可审核、可控制、可沉淀的项目型费用管理框架。

## 一、案例目标

- 预算前置
- 费用归项目
- 审核有依据
- 风险可识别
- 结果可核销
- 数据可沉淀

## 二、适用场景

适用于以“项目”为管理单位的费用场景，例如：

- 活动项目
- 市场项目
- 人事专项项目
- 行政采购项目
- 跨部门协同项目
- 其他有预算、有周期、有负责人、有费用归集要求的业务事项

## 三、整体结构

项目型费用管理体系按三个阶段设计：

```text
事前：预算建立与启用
-> 事中：费用发生与审核控制
-> 事后：结算、核销与归档
````

## 六、总体流转主线

```text
项目立项
-> 预算编制
-> 预算审核
-> 预算启用
-> 费用发生
-> 发票收集
-> 报销草稿
-> 项目关联
-> 费用分摊
-> 业务审核
-> 项目审核
-> 财务审核
-> 阶段性控制
-> 报销完成
-> 核销
-> 归档
-> 复盘
```

## 七、案例定位

这个案例用于展示：

* 项目型费用管理的基础逻辑
* 项目预算、项目归属、项目审核的控制思路
* 从费用发生到核销收尾的完整流程结构
* 项目费用管理中的风险控制与异常处理方式
