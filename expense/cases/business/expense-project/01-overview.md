# 总体设计

## 管理目标
- 预算前置
- 费用归项目
- 审核有依据
- 风险可识别
- 结果可核销
- 数据可沉淀

## 适用范围
- 活动项目
- 市场项目
- 人事专项项目
- 行政采购项目
- 跨部门协同项目

## 核心管理对象
- 项目
- 预算
- 费用
- 发票
- 报销单
- 审核节点
- 核销状态

## 管理口径说明
- 项目型费用管理以“项目”为归集单位
- 费用先判断是否属于对应项目，再进入后续流转
- 审核不仅看单据和发票，还要看项目归属、阶段安排和预算执行情况
- 同一笔费用如涉及多个项目或多个阶段，必须明确分摊依据
- 项目结束后，费用要能完成核销、归档和复盘

## 总体流转主线
项目立项
-> 预算编制
-> 预算审核
-> 预算启用
-> 费用发生
-> 发票收集
-> 报销草稿
-> 项目关联
-> 费用分摊
-> 业务审核
-> 项目审核
-> 财务审核
-> 阶段性控制
-> 报销完成
-> 核销
-> 归档
-> 复盘

## 管理重点
- 费用是否属于该项目
- 是否在该项目预算和阶段安排内
- 是否存在跨项目、跨阶段分摊
- 项目结束后是否能完成闭环
