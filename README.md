# Process Assets

这是一个用于沉淀流程数字化、低代码系统设计和业务标准化经验的个人资产库。

我希望把过去在行政管理、人事管理、费控报销、内部流程梳理和低代码工具搭建中的经验，整理成可以展示、复盘和复用的数字资产。

## 我关注的问题

- 如何把混乱的业务流程梳理成标准流程
- 如何把人工经验转化为表单、字段、规则和审批流
- 如何识别非标准场景，并设计人工处理机制
- 如何用低代码工具提升组织内部协作效率

## 仓库内容

- `cases/`：流程案例，用来记录问题、判断、动作和结果
- `methods/`：流程问题分类和处理方法
- `templates/`：流程收集、检查、设计和案例沉淀模板
- `GOAL.md`：当前阶段的项目目标与路线图

## 命名规则

- 基础通用案例使用 `basic-xxx.md`
- 业务案例使用 `business-xxx.md`

## 项目方向

这个仓库会逐步整理成一个小型流程数字化工具包，服务于三个目标：

- 找工作时展示流程数字化和低代码业务分析能力
- 给自己复盘过去的工作经验
- 为未来做数字游民、远程协作或独立服务沉淀基础资产

## 阅读顺序

1. 先看 `GOAL.md`，了解这个仓库的目标和路线图
2. 再看 `methods/`，了解流程问题如何分类和处理
3. 然后看 `templates/`，了解如何收集、检查、设计和沉淀流程
4. 最后看 `cases/`，查看已经整理出来的流程案例
