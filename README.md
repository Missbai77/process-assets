# Process Assets

一个用于沉淀流程数字化、低代码业务分析和运营系统设计能力的个人流程资产库。

这个仓库不是单纯记录流程说明，也不是零散保存表格模板，而是把真实业务场景拆解成可以阅读、复用、展示和持续扩展的数字化资产。

核心目标是证明：

- 能从混乱业务中识别流程节点、责任边界和风险点
- 能把线下管理经验转成表单、字段、状态、审批和规则
- 能用案例、方法和模板设计可落地的运营系统

---

## 这个仓库解决什么问题

很多企业内部流程的问题，不是没有制度，而是：

- 入口不清楚
- 责任边界不清楚
- 审批节点靠人记
- 数据无法追踪
- 异常场景没有标准处理方式
- 业务经验无法复用
- 流程上线后没有复盘数据

本仓库尝试把这些问题转成可配置、可追踪、可复盘的流程资产。

## 文档组织方式

本仓库采用：

专题
-> 案例
-> 方法
-> 模板

每个专题下尽量保持统一结构：

cases      真实或模拟业务案例
methods    流程拆解和判断方法
templates  可复用表单、清单和检查模板

案例文档通常按以下结构组织：

# 案例名称

## 背景
## 原流程痛点
## 标准化后的流程
## 表单字段设计
## 审批节点设计
## 状态流转
## 异常场景处理
## 可数字化部分
## 需要人工判断的部分
## 数据输出
## 复盘

## 能力展示

这个仓库主要展示以下能力：

### 1. 业务流程拆解能力

能从真实业务场景中拆出：

- 发起人
- 处理人
- 审批人
- 关键节点
- 异常节点
- 数据沉淀点

### 2. 低代码系统设计能力

能把流程转成：

- 表单字段
- 审批节点
- 状态流转
- 权限边界
- 提醒机制
- 数据报表

### 3. 运营系统搭建能力

能把单个流程扩展成一套可持续运行的系统：

- 案例沉淀
- 方法复用
- 模板复用
- 数据复盘
- 后续优化
