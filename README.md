# Process Assets

用于沉淀流程数字化、低代码系统设计和业务标准化经验的个人资产库。

这个仓库关注的不是单个表格或单个流程，而是把真实业务场景拆成可理解、可复用、可展示的数字化资产：案例、方法和模板。

## 仓库结构

```text
process-assets
-> expense/
   -> cases/
   -> methods/
   -> templates/
-> HR/
   -> cases/
   -> methods/
   -> templates/
-> GOAL.md
-> README.md
```

## 专题导航

### `expense/`

费用管理专题。

用于展示费用流程标准化、预算前置、过程控制、项目费用管理、费用复核和风险控制能力。

当前内容包括：

- `expense/cases/`：费用管理案例
- `expense/methods/`：费用管理方法
- `expense/templates/`：费用管理模板

### `HR/`

人力资源运营专题。

用于展示从用人需求、招聘、人效评估、人才库、面试、入职、入转调离、薪酬福利绩效到数据报告的完整 HR 运营系统设计能力。

当前内容包括：

- `HR/cases/`：HR 运营案例
- `HR/methods/`：HR 流程设计方法
- `HR/templates/`：HR 表单和检查模板

## 当前重点

当前仓库已经从单一费用专题，扩展为“费用管理 + 人力资源”两个专题。

下一阶段重点是补齐 HR 专题，尤其是：

- `HR/cases/hr-operating-system/`
- `HR/cases/employee-lifecycle/`
- `HR/cases/payroll-and-cost/`
- `HR/methods/`
- `HR/templates/`

## 文档写法

案例文档通常按以下结构组织：

```md
# 案例名称

## 背景
## 原流程痛点
## 标准化后的流程
## 表单字段设计
## 审批节点设计
## 状态流转
## 异常场景处理
## 可数字化部分
## 需要人工判断的部分
## 数据输出
## 复盘
```

方法文档用于沉淀判断逻辑，模板文档用于沉淀可复用表单、清单和检查项。
