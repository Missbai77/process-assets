# Cases

这个文件夹用来沉淀流程案例。

案例分为两类：

- 基础通用案例：使用 `basic-xxx.md` 命名，记录常见流程问题类型，例如信息通道、责任节点、规则口径、系统承接
- 业务案例：使用 `business-xxx.md` 命名，记录更接近真实业务场景的流程设计，例如费控报销、人事入职、行政采购等

## 基础通用案例

- `basic-info-channel.md`：信息通道问题
- `basic-responsibility-node.md`：责任节点问题
- `basic-rule-standard.md`：规则口径问题
- `basic-system-implementation.md`：系统承接问题

## 业务案例

- `business-expense-reimbursement.md`：费控报销流程案例

## 案例写作原则

- 不写真实公司、真实人员和敏感业务信息
- 尽量写清楚背景、痛点、判断、动作和结果
- 能系统化的部分写成字段、节点和规则
- 不能系统化的部分说明为什么需要人工判断
