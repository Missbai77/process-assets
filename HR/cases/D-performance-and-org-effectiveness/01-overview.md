# 总体设计

## 管理目标与原则

- 绩效口径统一
- 个人、部门、项目三类评价分开看
- 招聘需求不能只凭主观判断
- 编制逻辑要有数据支撑
- 绩效结果要能反向支持用人和组织调整
- AI时代下，绩效不能只看动作量和忙碌程度
- 结果、效率、复用、协同和判断能力要纳入评价
- 绩效结果必须可追溯、可复盘、可联动

## 核心对象

- 个人绩效结果
- 部门绩效结果
- 项目绩效结果
- 招聘需求
- 编制信息
- 人员投入情况
- 组织效能结果
- 绩效评价依据

## 流程主线

```text
绩效口径确认
-> 个人绩效评估
-> 部门绩效评估
-> 项目效能评估
-> 招聘需求有效性判断
-> 编制逻辑校验
-> 结果复盘
