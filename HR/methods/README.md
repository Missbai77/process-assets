# Methods

用于沉淀 HR 管理中的方法、判断逻辑与流程设计原则。

methods 不负责描述具体业务案例，而是用于统一：

- 管理口径
- 判断逻辑
- 流转逻辑
- 风险控制思路
- 流程设计原则

## 方法层特点

methods 强调：

- 为什么这样设计
- 如何判断
- 如何统一口径
- 如何控制风险
- 如何形成标准化流程

而不是具体业务案例。

---

## 与 cases 的区别

### methods

用于沉淀：

- 方法
- 原则
- 判断逻辑
- 管理思路

### cases

用于沉淀：

- 实际业务场景
- 表单
- 字段
- 审批节点
- 状态流转
- 风险场景
- 数据输出
