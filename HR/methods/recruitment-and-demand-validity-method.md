# 招聘需求有效性与招聘衔接方法

## 方法目标

- 让招聘需求提出有依据
- 让招聘执行有统一口径
- 让招聘结果可分流、可留痕
- 让录用结果与入职流程不断点衔接

## 方法主线

```text
需求提出
-> 需求信息确认
-> 有效性判断
-> 招聘执行
-> 面试评估
-> 结果分流
-> 录用衔接
-> 入职准备
```

## 核心判断逻辑

### 招聘需求是否真实成立

- 是否真实缺人
- 是否属于短期业务波峰
- 是否已有同类岗位
- 是否存在职责重叠
- 是否可通过流程优化解决
- 是否可通过系统或自动化降低需求
- 是否符合当前编制
- 是否有预算支持

### 招聘执行是否统一

- 岗位名称
- 岗位职责
- 招聘渠道
- 面试标准
- 面试记录
- 状态流转
- 录用标准

### 面试结果是否可分流

- 通过
- 不通过
- 进入人才库
- 暂缓录用
- 后续跟进
- 推荐其他岗位

### 录用结果是否能与入职衔接

- 入职时间
- 部门
- 岗位
- 汇报关系
- 薪资
- 权限
- 档案
- 入职资料

## 方法特点

- 招聘前置到需求判断
- 招聘与人效、编制、预算联动
- 招聘结果全过程留痕
- 招聘流程与员工生命周期衔接
- 支持后续组织效能分析

## 适用场景

- 常规招聘管理
- 编制控制场景
- 人效优化场景
- 多部门招聘协同
- 组织快速扩张阶段
- 流程数字化建设场景
