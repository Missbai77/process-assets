# 员工生命周期管理方法

## 方法目标

- 让员工状态全周期可追踪
- 让员工信息、权限、档案同步更新
- 让入转调离全过程可留痕
- 让员工状态与组织管理联动

## 方法主线

```text
录用确认
-> 入职准备
-> 入职办理
-> 试用期跟踪
-> 转正评估
-> 在职异动
-> 调岗调薪
-> 离职交接
-> 档案归档
```

## 核心管理逻辑

### 入职信息是否完整

需要确认：

- 身份信息
- 合同信息
- 岗位信息
- 汇报关系
- 部门归属
- 系统账号
- 权限配置
- 档案建立

### 试用期是否可跟踪

需要明确：

- 试用期目标
- 跟踪周期
- 评估标准
- 风险预警
- 转正条件

### 员工异动是否同步更新

员工状态变化后需要同步：

- 部门
- 岗位
- 汇报关系
- 权限
- 薪资
- 组织结构
- 档案信息

### 离职是否完整收口

需要确认：

- 工作交接
- 权限回收
- 资产归还
- 档案归档
- 离职状态更新

## 方法特点

- 员工状态全过程留痕
- 权限与员工状态联动
- 档案随生命周期同步更新
- 入转调离统一管理
- 支持后续组织与人效分析

## 适用场景

- 常规员工管理
- 快速扩张团队
- 多部门协同组织
- 流程数字化建设
- 低代码系统建设场景
